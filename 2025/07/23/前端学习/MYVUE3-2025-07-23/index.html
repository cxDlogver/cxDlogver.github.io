【Form规则校验】

登录注册页面通过在 Vue 中做表单规则校验，通常会使用 UI 框架（如 Element Plus、Naive UI、Ant Design Vue）内置的 表单验证机制。

  1. el-form的rules属性渲染规则。
  2. el-form的prop属性根据值加载规则。

主要分为三部分。

1. 校验规则

Element Plus 的 rules 是基于 async-validator 实现的，所以校验规则属性来源于它。

属性名 类型 说明
required boolean 是否必填
message string 校验失败时提示的信息
trigger `’blur’ ‘change’`
min number 字符串或数组的最小长度
max number 字符串或数组的最大长度
type string 类型校验，如 string、number、boolean、array、email 等
pattern RegExp 正则表达式
len number 字符串或数组的长度必须等于该值
enum array 值必须为列举中的某一个
whitespace boolean 是否允许全空格字符串（一般用于字符串 required: true 时控制空格）
validator Function AsyncFunction
fields object 嵌套对象的字段校验规则（多用于嵌套表单对象）
defaultField object 用于数组中的每一项校验

菜单设计

知识点总结

1.icon图标显示必须先全局引入所有Element Plus图标

2.菜单配置项中path属性必须以’/‘开头, 如果不是’/‘开头，会替换当前url的最后一个

例如 当前url为 /acl/usr, 菜单配置项中path属性为 ‘home’, 则会替换为 /acl/home

3.菜单配置项中path属性如果以’/‘开头，会直接替换当前url

例如 当前url为 /acl/usr, 菜单配置项中path属性为 /home, 则会替换为 /home

4.重点了解如何将NavMenu菜单项根据Routes路由项展示。

5.<component :is="item.children[0].meta.icon" /> 动态组件

6.递归调用自身,需要定义组件名,否则会报错

7.ref和reactive定义对象和数组的区别
