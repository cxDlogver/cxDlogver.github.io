面向对象编程

一、什么是面向对象编程？JS 的面向对象有什么特点？

口述模板：

面向对象编程是一种编程思想，它把程序看作是一组对象之间相互协作、传递信息的过程。
对象封装了事物的行为和数据，其中行为对应方法，数据对应属性，程序的执行过程就是对象之间调用方法、修改状态的过程。

JavaScript 也是支持面向对象编程的，但它和 Java、C++ 这类传统语言不同。
JavaScript 本质上是基于对象和原型的面向对象编程方式，而不是基于类复制实例。
ES6 引入的 class 只是语法糖，底层依然依赖原型机制。

所以理解 JS 面向对象的关键，不在于会不会写 class，而在于是否理解对象、原型以及原型链。

二、JavaScript 中创建对象的方式有哪些？

口述模板：

JavaScript 创建对象主要有四种常见方式。

第一种是对象字面量，适合创建结构简单、一次性使用的对象；
第二种是构造函数配合 new 关键字，用于批量创建结构相同的对象；
第三种是 Object.create，可以显式指定对象的原型，更贴近原型继承的本质；
第四种是 ES6 的 class 写法，它是对构造函数和 prototype 的封装，让代码结构更清晰、更易维护。

在工程中，class 更常用于表达对外的抽象结构，但底层原理仍然是原型。

三、什么是原型？原型链是如何工作的？

口述模板：

在 JavaScript 中，每一个对象在创建时都会关联一个内部原型，用于实现属性和方法的共享。
当访问对象的某个属性或方法时，如果对象自身没有，JS 引擎就会到它的原型对象上查找，这个过程叫做原型链查找。

原型对象本身也是对象，它也有自己的原型，于是层层向上，就形成了一条原型链，直到到达最顶层的 Object.prototype，它的原型是 null，查找结束。

原型链的意义在于：
让方法只需要定义一份，就能被多个实例共享，这是 JS 面向对象在性能和内存上的重要优势，也是 JS 实现继承的基础。

四、JavaScript 是如何实现继承的？class 继承的本质是什么？

口述模板：

JavaScript 的继承本质上是原型的继承。

在 ES6 之前，继承通常是通过修改原型链实现的，比如让子构造函数的 prototype 指向父构造函数的实例或原型对象。
ES6 的 class extends 只是让这种写法更直观，并没有改变底层机制。

class 继承本质上做了两件事：
一是让子类实例的原型指向父类的 prototype；
二是让子类构造函数本身的原型指向父类构造函数。

因此，class 继承只是对原型继承的一层语法封装。

五、JavaScript 是否支持封装、继承和多态？

口述模板：

从能力上讲，JavaScript 是支持面向对象三大特性的，但实现方式比较灵活。

封装可以通过作用域和闭包实现，在 ES6 之后，也可以通过 class 的私有字段来实现更清晰的封装；
继承依赖原型链实现；
多态则体现在“同一接口，不同实现”上。

JavaScript 的多态是典型的鸭子类型体现，它更关注对象“能做什么”，而不是对象“属于什么类型”。
只要不同对象实现了同名方法，就天然支持多态，而不依赖显式类型声明。

六、this 在面向对象中的指向规则是什么？

口述模板：

this 是 JavaScript 面向对象中非常容易出问题的点，因为它是在调用时确定的，而不是定义时。

作为对象方法调用时，this 指向调用该方法的对象；
作为普通函数调用时，在非严格模式下指向全局对象，在严格模式下是 undefined；
作为构造函数调用时，this 指向新创建的实例对象；
通过 call、apply、bind，可以显式指定 this 的指向。

很多 this 的问题，本质上都是调用方式发生了变化。

七、如何理解 JS 中的构造函数和构造器体系？

口述模板：

在 JavaScript 中，构造函数本质上也是普通函数，只是当它通过 new 调用时，承担了“创建对象”的角色。

Function 是所有函数对象的构造器，
Object 是普通对象的构造器。
Function 本身既是函数，又是对象，它是由自己构造出来的；
而 Object 也是函数，也是对象，它是由 Function 构造出来的。

Object.prototype 是所有普通对象的最顶层原型，
Function.prototype 是所有函数对象的原型，包括 Object 和 Function 本身。

这套关系体现了 JavaScript 构造器体系的自举特性。

八、new 关键字到底做了什么？

口述模板：

new 调用构造函数时，主要做了四件事：
首先创建一个空对象；
然后将这个对象的原型指向构造函数的 prototype；
接着把构造函数内部的 this 绑定到这个新对象；
最后返回这个对象。

这也是为什么构造函数和普通函数的最大区别，在于是否通过 new 调用。

九、方法定义在实例上和原型上有什么区别？

口述模板：

如果方法定义在实例对象上，那么每个实例都会拥有一份独立的方法，占用更多内存；
如果方法定义在原型对象上，那么所有实例共享同一份方法，内存利用率更高。

表达式方式和箭头方式定义的函数会定义在实例对象上。

因此在工程实践中，共享行为通常定义在原型上，实例只保存状态数据，这是 JS 面向对象的常见设计方式。
