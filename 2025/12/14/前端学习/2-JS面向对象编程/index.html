1.对象和函数

对象是对现实事物的一个抽象概述，面对对象编程就是将所有的程序封装成一个个对象,程序的运行过程就是一系列对象之间相互协作、消息传递的过程。

对象包括 属性和方法，表示事物的状态和动作。

每个对象都有一个内部属性 [[Prototype]]（通常访问为 __proto__），它指向另一个对象，这个对象称为“原型”。当访问对象的某个属性时，如果对象本身没有这个属性，则会去它的原型对象上查找，这就是“原型链查找”机制。原型本身也是对象，也有原型，因此构成一个原型链，直到最顶层null终止.

【Object和Function】

Object和Function是构造器，而构造器本身也是一个实例对象。

如何理解：

  • Object 是由 Function 构造出来的。Function 是由自己构造出来的。
  • Object.prototye是所有对象的顶层原型，Object.prototype.__proto__ === null。
  • Function.prototype 是所有函数对象的原型（包括 Object 和 Function 自身）
  • 函数本身也是对象，所以顶层原型在函数对象的原型链上。
对象 说明
Function 所有函数的构造器
Object 普通对象的构造器
Function 本身 也是函数 → 也是 Function 的实例
Object 本身 也是函数 → 也是 Function 的实例

函数对象和普通对象通过一些属性来区分，比如typeof 内部其实就是检测对象是否有 [[Call]]。

在 JS 里，一个对象是不是某个构造器的实例，只看一件事：

这个构造器的 prototype 是否出现在该对象的原型链上

也就是 instanceof 的本质。

2.构造函数和普通函数

  • 构造函数：除了有 [[Call]]，还具备 [[Construct]] 内部方法，所以能用 new。
  • 普通函数：如果没有 [[Construct]]，用 new 会报错。

【创建方式】

只有function和class创建的函数对象才有Construct构造器。

【constructor 属性】

  • constructor是区分 构造器函数 和 普通函数 的标准。

  • 构造器函数在创建时，都会自动获得一个 prototype 属性，指向一个原型对象。该原型对象上默认有一个 constructor 属性，指向函数本身。

  • 实例对象通过原型链可以访问到这个 constructor。

  • 普通函数没有prototype和constructor，但依然是实例对象，所有具备__proto__

【构造函数】

  1. 构造函数是用 new 调用的函数，可以创建对象。
  2. 类 (class) 是构造函数的语法糖，本质上还是函数。
  3. Function 是所有函数的构造器，它本身也是函数（自举特性）。
    • Function instanceof Function → true
    • Function instanceof Object → true
    • Object instanceof Function → true
