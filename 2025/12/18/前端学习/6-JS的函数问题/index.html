1.立即执行函数（IIFE）

在 JavaScript 模块化机制尚未完善之前，立即执行函数是一种常用的封装手段，用来避免变量污染全局作用域。其核心思想是：创建一个只执行一次的私有作用域，把变量和方法包裹在里面。

立即执行函数（Immediately Invoked Function Expression，IIFE）本质上是一个函数表达式。通过外层括号将函数声明转换为表达式后，紧随其后的 () 会使其在定义后立刻执行。由于它是表达式，因此可以有返回值，并将结果赋给变量。

在这段代码中，IIFE 只执行一次，创建了一个私有作用域，变量 count 只存在于该作用域内部。函数执行完成后返回一个对象，并赋值给 Counter。从外部只能通过 increment 和 reset 这两个方法操作 count，而无法直接访问或修改它，从而实现了模块化封装。

2.闭包（Closure）

在上述例子中，count 并没有随着 IIFE 的执行结束而被销毁，increment 和 reset 仍然可以持续访问它。这正是闭包的体现。

闭包的本质是：函数可以访问其定义时的词法作用域，即使该函数在其定义作用域之外被调用。只要某个内部变量仍然被外部函数引用，JavaScript 引擎就不会回收对应的作用域。

在 Counter 示例中，increment 和 reset 方法都引用了 count，因此该变量会被长期保留在内存中，但对外部代码不可见。这使得 count 成为一种“私有状态”，只能通过受控的方式被修改。

如果从类的角度类比理解，可以认为：类的方法可以访问私有字段，而实例对象本身不能直接访问这些字段。闭包在函数层面实现了类似的封装能力，是 JavaScript 早期实现私有变量和模块化的核心机制。

立即执行函数用于创建独立作用域并封装模块，而闭包则保证了该作用域在函数执行结束后仍能被安全地访问和维护。二者结合，构成了 JavaScript 在模块化与私有变量出现之前最重要的设计模式之一。

3.this

this 并不是 JavaScript 的语法关键字参数，而是函数在执行时由调用方式动态确定的隐含参数。理解 this 的关键，不在于“它是什么”，而在于函数是“如何被调用的”。

（3）call、apply、bind 的作用与区别

call、apply、bind 都用于显式指定函数执行时的 this 指向，区别在于是否立即执行以及参数传递方式。

方法 作用 是否立即执行 参数形式
call 改变 this 并执行函数 是 逐个参数
apply 改变 this 并执行函数 是 数组参数
bind 改变 this，返回新函数 否 逐个参数

需要注意的是：

apply 中传入的数组，会被拆解为多个独立参数，其行为与 ES6 的展开运算符 ...arr 等价。

（4）类中方法、函数表达式与箭头函数的区别
